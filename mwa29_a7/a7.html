Interactive Computer Graphics: Assignment 7

Bezier Patch

Topics:

  • Bezier Patches: Drawing a bicubic Bezier patch from a set of given control points
  • Interaction: User has the ability to interactively change the location of a control point and the resolution of the patch tessellation

Create a program that draws a smooth-shaded bicubic Bezier patch to the screen.
The initial control points for the patch are provided in a text file.
The file format will be 3 floating point numbers per line, with 16 lines.
X-Y Ordering: [0,0], [1,0], [2,0], [3,0], [0,1], [1,1], [2,1], etc.

Tessellate the patch into triangles and calculate a surface normal for each triangle vertex.
Initially the patch should be sampled with a 10 x 10 resolution.

You will draw the patch and the control points (as GL_POINTS) for the patch.

Also draw the X, Y and Z axes.

INTERACTION:

The user should be able to change the view of the patch.

The user should have the capability to choose a control point to modify, via keyboard interaction.
Change the color and/or size of the selected control point.
The user can then change the location of the selected point via 6 keyboard buttons (increase/decrease x/y/z position).

The user should be able to increase and decrease the sampling of the patch with a keyboard interaction.

The user should also be able to switch between smooth (using the Phong model) and flat diffuse-only shading.

Grading Scheme

  1. Reading in and storing control points (1 point)
  2. Tessellating patch (2 points)
  3. Smooth-shading (1 point)
  4. Drawing axes and control points (1 point)
  5. Changing view of patch (1 point)
  6. Selecting control point (1 point)
  7. Moving control point (1 point)
  8. Updating patch triangles (1 point)
  9. Modifying triangle resolution (1 point)
  10. Being able to switch to flat shading (1 point)
